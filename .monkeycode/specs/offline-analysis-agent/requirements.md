|------|------|

## 三、功能需求

### 3.1 Azkaban 任务生成（核心）

| 需求 ID | 功能点 | 详细描述 | 优先级 |
|---------|--------|----------|--------|
| FR001 | 任务类型 | 仅支持 Hive SQL 任务类型 | P0 |
| FR002 | 输入方式 | 支持自然语言 + 表单配置 + 模板参数组合输入 | P0 |
| FR003 | 预置模板 | 提供 7 大核心模板：单表聚合、多表关联、增量同步、全量同步、去重清洗、SCD、指标计算 | P0 |
| FR004 | 依赖配置 | 组合方式自动分析依赖（SQL 解析 + 元数据查询 + 用户配置） | P0 |
| FR005 | 分区处理 | 用户手动指定分区参数 | P0 |
| FR006 | 增量/全量 | 用户手动选择同步方式 | P0 |
| FR007 | Azkaban 配置 | 支持资源队列配置 | P1 |
| FR008 | 对接方式 | 通过 Azkaban REST API 上传和执行任务 | P0 |
| FR009 | 执行流程 | GUI 测试执行 → 人工审核 → 自动上传 | P0 |
| FR010 | 错误处理 | SQL 语法错误自动重试修正 | P1 |
| FR011 | 异常处理 | Hive 查询超时处理 | P1 |

### 3.2 元数据管理

| 需求 ID | 功能点 | 详细描述 | 优先级 |
|---------|--------|----------|--------|
| FR012 | 元数据来源 | 直连 MySQL 元数据库（Hive Metastore 后端） | P0 |
| FR013 | 同步方式 | 手动触发同步 | P1 |
| FR014 | 缓存策略 | 元数据本地缓存 | P1 |
| FR015 | 知识库建设 | 表关系（最优先）、指标定义、SQL 案例、FAQ、数据质量标准、命名规范 | P1 |

### 3.3 PC GUI 界面

| 需求 ID | 功能点 | 详细描述 | 优先级 |
|---------|--------|----------|--------|
| FR016 | 技术栈 | Rust + gpui 框架 | P0 |
| FR017 | 认证方式 | 标准 LDAP 对接 | P1 |
| FR018 | 国际化 | 多语言可扩展 | P2 |
| FR019 | 创建向导 | 分步表单引导用户创建任务 | P0 |
| FR020 | 任务管理 | 任务列表、搜索、编辑、删除 | P0 |
| FR021 | DAG 可视化 | 任务依赖关系图形化展示 | P0 |
| FR022 | SQL 编辑器 | 支持高亮和智能提示 | P0 |
| FR023 | 执行历史 | 查看任务历史执行记录 | P1 |
| FR024 | 日志查看 | 查看任务执行日志 | P1 |
| FR025 | 数据预览 | Hive 表前 100 行预览 | P1 |
| FR026 | 版本管理 | Git 集成（每任务一分支、自动 commit、人工 review、自动 push、自动 MR） | P1 |
| FR027 | 自动更新 | 检测更新 → 推送通知 → 用户重启安装 | P2 |

### 3.4 数据质量告警

| 需求 ID | 功能点 | 详细描述 | 优先级 |
|---------|--------|----------|--------|
| FR028 | 非空检查 | 检查字段是否为 NULL | P2 |
| FR029 | 唯一性检查 | 检查主键/业务键唯一性 | P2 |
| FR030 | 值域检查 | 检查数值是否在合理范围 | P2 |
| FR031 | 枚举值检查 | 检查值是否在枚举列表内 | P2 |
| FR032 | 波动检查 | 检查数据环比、同比波动 | P2 |
| FR033 | 行数检查 | 检查数据行数波动 | P2 |
